Launch Guide · $0 Budget

Build & launch Gather for free

Every tool you need has a free tier. Here's the complete stack — from domain to payments — without spending a dollar until you're making money.

1
Host the app for free
100% free
  • 🌐
    GitHub Pages (static hosting)

    Push this HTML file to a GitHub repo. Enable Pages in Settings → Pages → Deploy from main branch. Your app is live at yourname.github.io/gather in 2 minutes.

    → pages.github.com
  • ▲
    Vercel (best option — custom domain, HTTPS)

    Connect your GitHub repo to Vercel. Auto-deploys on every push. Free tier includes custom domain, SSL, analytics. Zero config needed.

    → vercel.com
  • 🔥
    Netlify (alternative)

    Drag-and-drop your folder into Netlify's UI. Live in 30 seconds. Free SSL and CDN. 100GB bandwidth/month free.

    → netlify.com
2
Backend & database (free)
100% free
  • 🗄️
    Supabase (free tier — Postgres + Auth + Storage)

    Create a free project. Use Supabase Auth for login (email/password, phone OTP). Use Postgres tables for hangouts, users, bookings. Free tier: 500MB DB, 1GB storage, 50K monthly active users.

    → supabase.com
  • ⚡
    Firebase (alternative — Google)

    Firestore + Firebase Auth. Free Spark plan: 1GB storage, 50K reads/day, 20K writes/day. More than enough to validate your MVP.

    → firebase.google.com
  • 🔑
    Phone verification — Twilio (free trial)

    Twilio gives $15 free trial credit for SMS OTP verification. That's ~100 verifications — enough to validate your first cohort. Then use Supabase's built-in phone auth (uses Twilio under the hood).

3
Payments with Stripe
No monthly fee
  • 💳
    Stripe — zero monthly cost

    Create a free Stripe account at stripe.com. No setup fee. No monthly fee. You only pay when you process money: 2.9% + 30¢ per transaction in Canada/US.

    → stripe.com
  • 🔑
    Get your Publishable Key

    Dashboard → Developers → API Keys → copy your pk_live_… key. Replace STRIPE_PK in this file's script tag. For testing use pk_test_… first.

  • ⚙️
    Backend: Stripe Payment Intents via Supabase Edge Function

    Create a Supabase Edge Function (free) that accepts amount + customer info, creates a PaymentIntent, and returns the client_secret. Your frontend then confirms with card details. Never expose your secret key in frontend code.

  • 💸
    Stripe Connect for host payouts (recommended)

    Use Stripe Connect to pay hosts directly. Hosts onboard their own bank account. You take a platform fee (10%) and Stripe transfers the rest. No float, no manual payouts.

4
Email, notifications & maps
Free tiers
  • 📧
    Resend — transactional email (free)

    3,000 emails/month free. Send booking confirmations, 24hr reminders, host notifications. Simple REST API or SDK.

    → resend.com
  • 🗺️
    Google Maps Embed — free

    Static map embeds and the Maps Embed API are free with no billing required. Show venue location on detail pages. Only the Maps JavaScript API has a $200/month free credit (which covers ~40K map loads).

  • 📱
    Push notifications — OneSignal (free)

    Free web push notifications up to unlimited subscribers. Send "Your hangout starts in 1 hour!" reminders. Zapier-free webhook integration with Supabase triggers.

    → onesignal.com
5
Get your first 200 users (no ad spend)
Week 1
  • 🎯
    Pick one city, one niche

    Don't launch everywhere. Pick Waterloo + "board games" OR "hiking". Create 3 hangouts yourself manually. Be the first host.

  • 👽
    Post in local Reddit subreddits

    r/waterloo, r/uwaterloo — share the hangouts as genuine community posts, not spam. "I built this thing because I moved here and couldn't make friends. Anyone want to try it?"

  • 📢
    Facebook Groups

    Search "Waterloo friends", "KW newcomers", "Board games Kitchener". Post your events there. Local Facebook groups have much higher engagement than you'd expect.

  • 🏫
    University & co-working notice boards

    UW and Laurier have physical + digital bulletin boards. Students are exactly your audience — post a QR code flyer. Cost: $0 at most libraries.

  • 📊
    Measure "second hangout rate"

    Track what % of attendees come to a second event. If that number is above 40%, you have product-market fit. Below 20%, change the format.
